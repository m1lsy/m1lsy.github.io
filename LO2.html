Development & Version Control

Brief reflection on past Portfolio

In regards to Development, this semester I have focused on improving my skills in HTML, CSS and JavaScript. Coding is something which I actually really enjoy and often find exciting, however I tend to make a lot of mistakes and I often struggle to grasp concepts as fast as I would like to, which can leave me feeling less motivated and discouraged.

To start the semester, I did a lot of reflecting on my previous Portfolio from semester one and focused on the problems that it had in order to fix them going forward. Figure A below shows a small snippet of CSS code from my old portfolio where many examples of the mistakes I made are evident. These mistakes include:

    -REPITITION: The font family has been declared many times which is unnecessary and can be easily avoided by defining a global font family.
    -MAGIC NUMBERS: The code uses hardcoded values for widths, height and font size which can make maintenance and stability a challenge.
    -LACK OF CONCISENCY: The code uses different units for fonts (px) and widths/heights (px and %). Its better practice to just use one unit of measurement throughout the code.
    -INCOMPLETE SELECTORS: 'H3' and 'H6' selectors are not specific to any class or ID which can result in unintended styling. Using a class and ID can help target specific design elements.
    -NO COMMENTS: Not commenting code is a bad practice as it will make it harder for developers to maintain, collaborate and understand my code.
    -CLASS NAMES: Another problem with my code which I picked up on by getting feedback from some other students was that my class names were often confusing and too lengthy.

It is very important to fix simple mistakes like these as they can lead to bad habits which will affect me later on, especially when collaborating with other people as they will face difficulties interpreting my code. With these aspects in mind, I began my developing process for my portfolio.



Sematic HTML

What I Did

Attending Metaxas' lectures taught me the importance of using semantic HTML. During this semester, I made a conscious effort to use semantic tags instead of defaulting to <div> or <span>, which I often use due to ingrained coding habits. For instance, I used the <section> tag to organize my page into distinct sections, which allowed me to change the background color of specific areas.

What I learned

I learned that semantic HTML is crucial for creating web content that is accessible, maintainable, and SEO-friendly. This awareness has highlighted the importance of using appropriate HTML tags rather than relying on non-semantic ones.

How I think it went

Reflecting on my progress, I recognize that in future projects, I need to focus more on reducing my reliance on the <div> tag. Despite my efforts, I still use it too frequently. Although it feels almost like muscle memory to reach for a <div> tag, I am determined to continue improving and fully adopt the use of semantic HTML.
